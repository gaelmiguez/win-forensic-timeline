# Política de seguridad

## Comunicación responsable

Comunique las vulnerabilidades de forma privada al propietario del repositorio mediante el canal de reporte de seguridad de GitHub cuando esté disponible. No publique detalles explotables antes de que el problema haya sido evaluado.

Incluya una descripción mínima, la versión afectada, pasos de reproducción con datos sintéticos y el impacto esperado. No adjunte credenciales, evidencias forenses reales, historiales, EVTX, hives, Prefetch, bases de datos, volcados ni outputs de una investigación.

## Alcance

El proyecto es un prototipo académico y no una herramienta certificada. Los hallazgos relacionados con integridad probatoria, adquisición, parsing, manejo de rutas, exportaciones o exposición de información sensible son especialmente relevantes.

## Datos sensibles

Si un informe requiere un ejemplo, sustituya todos los datos por fixtures sintéticas. Si se publica accidentalmente información sensible, notifíquelo de inmediato y no la replique en un issue público.
